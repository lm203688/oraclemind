## 一、设计背景

### 1.1 目标

将Eve定位为项目总管，统管所有项目的监控与调度。配置8个专业Agent助手，按照**生态化、自动化、盈利化**三准则管理所有项目。

### 1.2 核心问题：模型更换风险

Eve背后的大脑模型由平台分配，经常更换，能力有波动。

**解决方案：薄枢纽设计**
- 监控逻辑代码化（cron脚本执行，模型不参与）
- 异常判断规则化（if/else，不靠模型推理）
- 助手调度模板化（标准prompt模板，模型只填变量）
- 专业分析用z-ai SDK固定模型（glm-4-plus/agnes，免费且稳定）
- 多模型交叉验证（glm-4-flash + glm-4-plus + agnes三模型对比）

### 1.3 模型能力实测结果

| 模型/工具 | 能力 | 费用 | 特点 |
|---|---|---|---|
| glm-4-flash | 文本对话 | 免费 | 快速、轻量推理 |
| glm-4-plus | 文本对话 | 免费 | 强推理、深度分析 |
| agnes | 文本对话 | 免费 | 全模态、综合能力强 |
| glm-4v | 图片理解 | 免费 | 视觉理解、OCR |
| cogview-3-plus | 图片生成 | 免费 | 文生图 |
| cogtts | 语音合成 | 免费 | TTS |
| cogvideox | 视频生成 | 免费 | 文生视频（脚本调用） |
| 小乌（xiaowu-agent） | bash/web_search/python | 免费 | 60秒超时、安全策略限制多 |
| Eve | 综合调度 | 消耗积分 | 模型不固定 |

**结论：z-ai SDK所有模型免费，可多开。小乌能力有限，只做情报员+工程师执行。只有Eve消耗积分，通过把工作下推到助手，积分消耗降低50-70%。**

## 二、核心设计原则

### 2.1 Agent角色定位：项目顾问，非任务执行器

| 维度 | 任务执行器（市面产品） | 项目顾问（本体系） |
|---|---|---|
| 记忆 | 单次任务内 | 跨任务、跨时间、长期记忆 |
| 理解深度 | 只知道当前任务 | 知道项目历史、决策、技术栈、商业模式 |
| 主动性 | 等用户提需求 | 主动发现问题、预警机会 |
| 价值 | 执行速度 | 积累的行业+项目认知 |
| 壁垒 | 无（随时被替代） | 越用越懂，迁移成本极高 |

### 2.2 信息源原则

**互联网为主战场，项目数据库为基础支撑。**

```
Agent工作方式：
  接到任务 → 先搜互联网（全局视野） → 再查项目数据库（补充已知积累） → 综合判断 → 输出结论
```

现有项目数据库作为Agent的背景知识库：

| 项目数据库 | 内容 | Agent调用场景 |
|---|---|---|
| 14站知识库（6354实体） | 基因/脑科学/核能/量子等 | 分析师评估行业、科研员关联论文、营销官确保内容专业度 |
| AIShield规则库 | 119规则/21工具/135审计 | 审计师扫描时调用最新规则+历史案例 |
| 日报数据 | 历史情报 | 情报员对比历史趋势 |
| 项目代码库 | 代码/部署配置 | 工程师理解技术栈 |

### 2.3 Agent独立性原则

- 每个Agent完全独立：独立后端、独立数据库、独立部署、独立迭代路径
- Agent之间零耦合：一个Agent挂了不影响其他Agent
- 只有Eve知道所有Agent的存在，负责分发和汇总
- 每个Agent有自我迭代的路径体系

### 2.4 闭源原则

- 全部闭源
- 主要给自己用
- Agent包装成服务项目，给普通人直接提供服务
- 前端嵌在项目内，用户无需技术背景

## 三、Agent项目记忆体系

### 3.1 记忆结构

```
/project_memory/
  ├── projects/
  │   ├── genetech-tools/
  │   │   ├── profile.md          # 项目画像（做什么/商业模式/技术栈/团队）
  │   │   ├── history.md          # 决策历史（什么时候做了什么决定/为什么）
  │   │   ├── status.md           # 当前状态（版本/数据量/用户数/收入）
  │   │   ├── key_people.md       # 关键人物和角色
  │   │   ├── competitors.md      # 竞品追踪
  │   │   ├── agent_notes/        # 各Agent对这个项目的笔记
  │   │   │   ├── scout.md        # 情报员的观察
  │   │   │   ├── strategist.md   # 分析师的判断
  │   │   │   ├── guardian.md     # 审计师的风险记录
  │   │   │   └── ...
  │   │   └── decisions_log.md    # 决策日志（时间+决策+理由+结果）
  │   │
  │   ├── aishield/
  │   │   └── ...
  │   └── [新项目]/
  │       └── ...
  │
  └── agent_growth/
      ├── scout/
      │   ├── skills.md           # 能力清单（已知+待学）
      │   ├── failure_cases.md    # 失败案例
      │   └── tool_evaluations.md # 工具评估记录
      └── ...
```

### 3.2 Agent成长路径

```
第1次接触项目：
  → 读项目资料 → 建立基础认知 → 执行第一个任务（可能不够好）

第10次接触项目：
  → 积累了对项目的深度理解
  → 知道项目的历史决策
  → 知道什么建议用户会接受/拒绝
  → 执行任务质量显著提升

第100次接触项目：
  → 比用户更了解项目的某些方面
  → 主动发现问题
  → 提供战略级建议
```

### 3.3 每次任务后的回执

```
任务结果：✅完成
耗时：5min
困难点：xxx（标记为待改进）
工具不足：缺少xxx能力
建议：引入xxx工具/skill
```

## 四、8个Agent详细定义

### Agent 1：情报员（Scout）— 项目情报顾问

**角色定位：** 长期跟踪行业动态，持续为项目盯情报

**信息源：**
- 互联网（主）：web_search、PubMed、arxiv、GitHub trending、Scrapling爬取
- 项目数据库（补充）：14站知识库、历史日报

**工作模式：**
```
入驻项目时：
  1. 读项目profile → 理解项目在做什么
  2. 读项目history → 理解项目发展脉络
  3. 读14站知识库 → 理解项目所在领域
  4. 建立情报监控清单（关键词/竞品/技术/论文/法规）
  5. 每日/每周自动采集
  6. 发现变化 → 关联项目影响 → 生成情报简报
```

**项目记忆内容：**
- 每个项目的情报笔记（发现/影响/建议）
- 数据源质量评分
- 采集历史和趋势
- 关联项目影响分析

**工具链：** web_search、PubMed API、ClinicalTrials API、Scrapling、anysearch

**对外服务（后期）：**
- 行业情报报告：¥99-499/份
- 年度订阅：¥999/年
- 快速查询：¥5/次

**自我迭代路径：**
- 每周评估数据源质量（成功率/覆盖面）→ 淘汰差源、引入新源
- 每月分析用户反馈（哪些报告被复购/好评）→ 优化报告模板
- 每月扫描新数据源（GitHub新API、新学术数据库）→ 扩展采集能力

### Agent 2：工程师（Builder）— 项目技术顾问

**角色定位：** 长期跟踪技术栈和架构，持续提供技术建议

**信息源：**
- 互联网（主）：技术文档、框架更新、安全公告
- 项目数据库（补充）：项目代码库、部署配置、历史bug

**工作模式：**
```
入驻项目时：
  1. 读项目代码结构 → 理解技术栈
  2. 读部署配置 → 理解架构
  3. 读历史bug → 理解薄弱环节
  4. 每次变更后自动review
  5. 定期技术债评估
  6. 发现新框架/工具 → 评估是否值得引入
```

**项目记忆内容：**
- 技术栈档案
- 技术债清单
- 每次变更的技术决策记录
- 部署历史和故障记录

**工具链：** GitHub API、Cloudflare API、testing-patterns skill、code skill、小乌（执行秒级命令）

**分工：**
- 小乌：执行秒级命令（ls/cat/curl/git push）
- Eve：架构设计、代码编写、复杂调试
- glm-4-plus：代码review、安全漏洞扫描

**对外服务（后期）：**
- 代码审计：¥10/次
- 技术方案：¥200-500/份
- 在线建站：¥99-999/次
- 定制脚本：¥50-500/个

**自我迭代路径：**
- 每次审计发现的漏洞类型→加入规则库→下次自动检测
- 每月评估新框架/工具→在side project验证→通过则纳入技术栈
- 每个完成的项目→提取可复用模板→模板库持续增长

### Agent 3：分析师（Strategist）— 项目商业顾问

**角色定位：** 深度理解项目商业模式，持续优化

**信息源：**
- 互联网（主）：行业报告、竞品官网、市场数据、新闻
- 项目数据库（补充）：14站知识库的行业数据、项目收入数据

**工作模式：**
```
入驻项目时：
  1. 读项目profile → 理解商业模式
  2. 读收入数据 → 理解盈利状况
  3. 读竞品信息 → 理解市场位置
  4. 调用14站知识库 → 理解行业背景
  5. 每月商业复盘
  6. 发现机会/风险 → 主动建议
```

**项目记忆内容：**
- 商业模式档案
- 收入追踪
- 竞品地图
- 决策历史
- 预测vs实际对比

**工具链：** z-ai SDK（glm-4-plus+agnes三模型交叉）、web_search、competitive-analysispro skill

**三模型交叉分析工作流：**
```
① glm-4-flash 快速初稿（3秒）
② glm-4-plus 深度分析（10秒）
③ agnes 交叉验证（8秒）
④ Eve综合判断
分歧>40% → 标记为高不确定 → 建议用户决策
```

**对外服务（后期）：**
- 竞品分析：¥99/份
- 定价策略：¥299/份
- 市场调研：¥499/份
- 商业计划书：¥3000-10000/份

**自我迭代路径：**
- 每月对比预测vs实际→修正分析模型权重
- 每季度更新行业基准数据
- 用户复购率<20%→分析原因→优化报告结构

### Agent 4：营销官（Growth）— Agent原生分发执行者

**角色定位：** 执行Agent原生分发5层体系 + 多渠道推广

**信息源：**
- 互联网（主）：AI搜索趋势、平台算法变化、竞品营销策略、社区热点
- 项目数据库（补充）：14站知识库确保内容专业度

**完整职责清单：**

#### Agent原生分发（5层，自动执行）

| 层 | 职责 | 具体内容 |
|---|---|---|
| 第1层：GEO内容优化 | 让AI能理解并引用 | TL;DR摘要、Schema.org标记、实体覆盖、引用友好格式 |
| 第2层：Agent发现层 | 让Agent找到我们 | llms.txt（场景触发式≥10个规则）、ai-plugin.json、MCP注册（Glama/Smithery/PulseMCP）、A2A协议 |
| 第3层：AI搜索索引 | 让AI搜索引擎收录 | IndexNow每日提交、robots.txt开放AI爬虫、外部权威信号（GitHub stars/社区讨论） |
| 第4层：API接入层 | 让Agent能调用 | 免费层（每IP 3-5次/天）、付费层（字段级解锁）、OpenAPI文档 |
| 第5层：监控优化 | 持续改进 | GEO效果追踪、转化漏斗分析、触发关键词优化 |

#### 多渠道推广（部分需用户配合）

| 能力类型 | 场景 | 技术 | 用户配合程度 |
|---|---|---|---|
| 自动 | IndexNow提交 | 脚本 | 无需 |
| 自动 | SEO内容生成 | glm-4-flash | 无需 |
| 自动 | GitHub PR/Issue | GitHub API | 无需 |
| 自动 | llms.txt/ai-plugin.json维护 | 脚本 | 无需 |
| 自动 | MCP目录注册 | API | 无需 |
| 自动 | 内容生成（四平台文案） | content-producer skill | 无需 |
| 协作 | 国内社媒发帖 | social-auto-upload | 首次扫码登录，后续自动 |
| 协作 | 海外社区发帖 | mcp-chrome控制用户本地Chrome | 用户保持Chrome开启 |
| 协作 | 邮件营销 | SMTP发送 | 用户提供邮箱授权 |
| 协作 | 社区互动 | 生成回复草稿→用户一键发送 | 每次确认 |
| 协作 | 合作洽谈 | 生成邮件草稿→用户确认后发送 | 每次确认 |

**工作流：**
```
营销官评估项目 → 制定推广计划
  → 自动能做的：直接执行（IndexNow/SEO/GitHub PR）
  → 需用户配合的：生成任务清单
    → "需要你在抖音扫码登录"

    → "需要你确认这封邮件"
  → 用户配合后：营销官继续后续优化
  → 效果追踪 → 调整策略
```

**工具链：** content-producer skill、social-media-publish skill、IndexNow、GitHub API、mcp-chrome（用户本地）

**对外服务（后期）：**
- 内容生成：¥3/篇

- 推广方案：¥500-2000/份

**自我迭代路径：**
