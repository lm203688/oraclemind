# 视频脚本 3：《开源 vs 融了 $33M 的竞品》

**类型**: 叙事型 / 观点输出型  
**时长**: 60-90 秒  
**格式**: 截图展示 + 屏幕录制 + 字幕 + 旁白  
**目标**: 建立差异化认知，输出开源理念，吸引认同价值观的开发者长期关注  
**核心情绪弧线**: 震惊（竞品融资）→ 质疑（黑盒风险）→ 认同（开源才是正确方向）→ 行动

### 第 38-55 秒 | 我们的做法：开源才是正确答案

**画面**:  
切换到真实的 GitHub 页面（屏幕录制）。

打开项目的评分计算核心文件（模拟，展示代码逻辑）：

**展示内容**（选取直观的评分公式代码）：

```typescript
// 信任评分核心公式 - 完全开源，可审计
export function calculateTrustScore(metrics: AgentMetrics): TrustScore {
  const scores = {
    performance:  metrics.responseTime * 0.30,  // 30% 权重
    security:     metrics.incidentRate  * 0.25,  // 25% 权重
    reliability:  metrics.uptime        * 0.25,  // 25% 权重
    compliance:   metrics.standardPass  * 0.20,  // 20% 权重
  };
  
  const total = Object.values(scores).reduce((a, b) => a + b, 0);
  return { total, grade: toGrade(total), breakdown: scores };
}
```

屏幕上用红圈标注「0.30」「0.25」「0.25」「0.20」——公式权重一目了然。

用打字工具在屏幕上叠加文字：

```
这就是评分公式
每一行权重，公开透明
任何人都可以审计
```
