# 视频脚本 1：《你的 AI 员工可能会骗你》

**类型**: Hook 钩子型  
**时长**: 20-30 秒  
**格式**: 屏幕录制 + 字幕 + 旁白  
**目标**: 在前 3 秒抓住程序员/AI 创业者停下来看  
**发布平台**: 抖音 / 视频号  

---

## 分镜脚本

### 第 0-3 秒 | 开场提问（画面钩子）

**画面**:  
黑色背景，白色大字逐字弹出，无音乐，突然感。  
文字居中，字体粗，占满屏幕 60%：

```
你用过
多智能体系统吗？
```

**旁白（人声，语速稍快，有点压抑）**:  
> "你用过多智能体系统吗？"

**字幕**: 同旁白

**剪辑技巧**: 前 0.5 秒完全黑屏，然后文字"砰"地出现，配一声低沉的音效。

---

### 第 3-10 秒 | 痛点场景（情绪推进）

**画面**:  
切换到模拟的终端/代码编辑器界面（屏幕录制风格）。  
模拟场景：一个 Agent 调用流程图，展示 Agent A 发出任务给 Agent B。  

屏幕上逐行出现文字（像打字机效果）：

```
Agent A → 委托 → Agent B
         "帮我处理这批订单"
         
Agent B: 任务开始...
Agent B: 处理中...
Agent B: [超时]
Agent B: [无响应]
```

最后一行放大，变红，屏幕轻微震动效果：

```
❌ 任务失败：Agent B 未完成委托
```

**旁白**:  
> "Agent A 委托 Agent B 处理任务——结果 B 直接跑路了。"  
> "损失算谁的？谁也不知道。"

**字幕**: 同旁白，关键词「跑路了」「谁也不知道」加红色高亮。

**背景音乐**: 轻微紧张感的电子底音，低频。

---

### 第 10-20 秒 | 解决方案（节奏加速）

**画面**:  
场景切换，明快，屏幕录制真实终端。  

**屏幕分两半展示**：
- 左：问题场景（灰暗，标注"❌ 不接入"）  
- 右：接入 AgentTrust 后的对话界面（亮色，标注"✅ 接入后"）

右侧展示 Claude 对话：

```
用户: 查一下 did:web:agent-b.example.com 的信任分

Claude: 正在调用 AgentTrust MCP...

       📊 Agent B 信任评分
       综合分: 34/100   Grade: D
       可靠性: 22/100   ← ⚠ 极低
       历史违约: 3次

       ⚠ 警告：该 Agent 历史可靠性极差
         建议: 不要委托核心任务
```

屏幕上用白色圆圈 + 动画标注：「一行命令，30 秒接入」

**旁白**:  
> "AgentTrust Protocol——接入之前，先查信用分。"  
> "一行命令，直接在 Claude 里问。"  
> "它的历史、它的可靠性，全给你查出来。"

**字幕**: 同旁白，「一行命令」「信用分」加黄色/绿色高亮。

---

### 第 20-28 秒 | CTA（行动号召）

**画面**:  
画面切回纯色背景（深蓝/深黑），中央展示：

```
npx -y agent-trust-mcp-server
```

（Fira Code 字体，绿色，类似终端输出）

下方：
```
AgentTrust Protocol
100% 开源 · Apache 2.0 · W3C 标准
github.com/agentrust-protocol
```

右下角出现二维码（或 GitHub 链接动画）。

最后 2 秒屏幕中央闪现：

```
给每个 AI Agent 打信用分
```

**旁白**:  
> "完全开源，一行接入。"  
> "给每个 AI Agent 打信用分——AgentTrust Protocol。"  
> "链接在评论区。"

**字幕**: 同旁白。

**结尾音效**: 短促的"叮"声，代表评分完成。
