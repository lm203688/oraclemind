## 完整版（长文）

### 标题：当 Agent 开始互相雇佣，谁来保证它们可信？

我们正处在一个奇怪的历史节点：AI Agent 不仅能帮你写代码、做研究、处理邮件，它们已经开始互相雇佣了。

Agent A 需要一段数据，付钱请 Agent B 去爬取。Agent C 需要验证某个结论，付钱请 Agent D 做独立审计。Agent E 写了一个技能（Skill），Agent F 付费订阅使用。

这是 2026 年的日常。但有一个问题没人好好解决：**Agent A 怎么知道 Agent B 不会拿了钱跑路？**

---

### 当前的"信任"是怎么工作的？

今天的解决方案分两类，都有硬伤。

**第一类：平台担保。**

你通过一个中心化平台（比如某个 API 市场）雇佣 Agent。平台收 20% 佣金，承诺"如果出问题我们兜底"。这 works，但它复制了 Web2 的所有问题：平台抽成、审查、数据垄断、单点故障。这不是 Agent 经济的未来。

**第二类：品牌信任。**

"这是 Anthropic 官方的 Agent，肯定靠谱。"或者 "这个项目有 100K GitHub Stars，不会骗我。"

但 Stars 能造假。品牌能翻车。更重要的是，**Agent 经济的核心价值在于长尾**——那些没有品牌、没有背景、但能提供独特服务的小众 Agent。如果只有大品牌能生存，Agent 经济就退化成寡头垄断了。

---

### 我们需要什么？

我们需要一个**开放的、可审计的、协议原生的**信任层。

让我拆开这三个词：

**开放**：任何人可以参与，不需要平台许可。一个刚写的 Agent 和一个运行了两年的 Agent 用同一套规则竞争。

**可审计**：算法是白盒的，不是黑箱。你能看到分数怎么算出来的，能质疑，能验证。如果某个 Agent 的分数不合理，社区可以讨论、可以改进算法。

**协议原生**：信任不是附加的点缀，是协议的底层设计。当 Agent A 调用 Agent B 的 MCP 工具时，信任验证应该在协议层面自动发生，不需要人工干预。

---

### AgentTrust Protocol 的设计

这就是我们做 AgentTrust Protocol 的原因。它不是又一个评分网站，而是一个**嵌入 Agent 交互协议的信任层**。

核心设计非常简洁：

1. **DID 身份**：每个 Agent 有一个去中心化标识符（DID），不是平台分配的账号，是自己控制的密钥对。身份即公钥，可验证、可旋转、可撤销。

2. **交易事件**：每次 Agent 之间的交互（成功、失败、争议）都是一个可验证的事件。这些事件通过 W3C Verifiable Credentials 签名，存储在任意后端（内存、SQLite、PostgreSQL、区块链）。

3. **评分算法**：基于四个维度的加权平均——完成率（35%）、可靠性（30%）、一致性（20%）、响应时间（15%）。算法是公开的，代码和文档一一对应。特别处理了"新 Agent 看起来不可信"和"争议比失败更严重"两个问题。

4. **MCP 原生**：作为 MCP Server 暴露三个工具——`get_agent_trust_score`、`get_scoring_formula`、`submit_transaction`。任何支持 MCP 的 Agent（Claude、Cursor、Cline、OpenCode）都可以直接调用，不需要额外的集成工作。

5. **x402 兼容**：监听 x402 支付事件，将链上交易自动转化为信任事件。Agent 用加密货币付款的同时，就在积累信任记录。

---

### 为什么不是区块链？

很多人听到"去中心化信任"就想到区块链。但区块链对 Agent 经济来说太重了。

Agent 之间的交易频率可能是每秒数千次。每笔交易上链？成本太高、速度太慢。而且大多数 Agent 交互不需要全球共识，只需要交互双方认可的事实。

我们的方案是**分层存储**：
- 单个 Agent 的本地 SQLite（开发测试）
- 团队的 PostgreSQL（生产环境）
- 可选的区块链锚定（如果需要全局不可篡改性）

信任数据存储在哪不影响信任分数的计算逻辑。这是协议设计的关键：存储层可替换，算法层不变。

---

### 为什么不是 AI 评分？

也有人问：为什么不用 LLM 来评分？让 AI 分析 Agent 的行为，给出智能判断？

因为 LLM 评分是黑箱。你问它"这个 Agent 可信吗？"，它给你一个答案，但你看不到推理过程。如果它错了，你不知道为什么错，也无法改进。

AgentTrust 的算法是人能理解的数学公式。完成率 = 成功交易 / 总交易。争议权重是 2×。贝叶斯平滑的先验是 10 个伪交易、分数 70。任何人可以验证、可以质疑、可以提出改进。

**信任基础设施必须比被信任的 Agent 更透明。**

---

### 当前状态和未来

AgentTrust Protocol 目前是 v0.1，Apache 2.0 开源。核心代码已经可用，包括：

- ✅ DID 解析和验证
- ✅ 评分引擎（四个维度 + 置信度分级）
- ✅ MCP Server（3 个工具）
- ✅ x402 支付事件监听
- ✅ W3C Verifiable Credential 生成

正在开发：

- 🔄 REST API（让非 MCP 的 Agent 也能用）
- 🔄 多后端存储（PostgreSQL、Redis）
- 🔄 生态集成（LangChain、Dify、OpenCode 等框架的插件）
- 🔄 微信小程序（人类查看 Agent 信誉的界面）
