## 二、7个具体可参考点

### 1. AutoResearchClaw的23阶段流水线（参考架构）

AutoResearchClaw把科研分成8阶段23步：
```
Phase A: 研究定义(1-2)     Phase E: 实验执行(12-13)
Phase B: 文献(3-5)         Phase F: 分析(14-16)
Phase C: 假设(6-8)         Phase G: 写作(17-20)
Phase D: 实验设计(9-11)    Phase H: 评审(21-23)
```

**蜂群当前7种蜂的粒度太粗。** 可参考AutoResearchClaw的阶段设计，把蜂的职责细化为：
- 每种蜂对应明确阶段而非功能域
- 蜂之间有明确的数据流和交付物

**但不要照搬23步** — 蜂群优势是简洁，建议保持7蜂但加阶段标记。

### 2. AutoResearchClaw的Co-Pilot模式（参考交互）

6种人机协作模式：
- `full-auto` 全自动
- `gate-only` 仅关键节点确认
- `checkpoint` 检查点模式
- `step-by-step` 逐步
- `co-pilot` 副驾驶
- `custom` 自定义

**蜂群当前只有全自动模式。** 可参考加：
- 免费用户：`gate-only`（关键节点确认，增加信任感）
- 付费用户：`co-pilot`（深度参与，科研质量更高）

### 3. ARIS的跨模型对抗评审（参考质量保证）

ARIS核心创新：**Claude执行 → GPT-5.5评审**，对抗而非自审。
- 4轮自动评审循环，5/10 → 7.5/10过夜
- 79个可组合技能
- Research Wiki持久化记忆

**蜂群的评审蜂当前是LLM自审。** 可参考：
- 评审蜂用不同模型评审写作蜂的输出
- 多轮迭代而非单次
- ⚠️蜂群用agnes免费模型，跨模型需要多API，成本上升

### 4. ARIS的飞书集成（参考分发渠道）

ARIS支持飞书通知和交互：
- Push模式：群聊卡片（实验完成/检查点/错误）
- Interactive模式：私聊与Claude Code交互（批准/拒绝/自定义指令）

**蜂群挂载在AIShield gunicorn上，已有飞书基础。** 可直接参考ARIS的飞书交互模式，让用户在飞书里：
- 接收蜂群进度推送
- 在飞书里批准/拒绝实验方案
- 飞书群作为社区

### 5. AutoResearchClaw的Skill加载系统（参考扩展性）

支持加载开源和自定义技能：
- `researchclaw skills install`安装技能
- 放`SKILL.md`到`.claude/skills/`即可
- 20个预装技能（科研写作/实验设计/化学/生物）
- 社区贡献技能（如A-Evolve进化技能）

**蜂群当前7种蜂是硬编码。** 可参考：
- 蜂定义为Skill（SKILL.md格式）
- 支持用户/社区贡献蜂
- 化学蜂/生物蜂等作为可安装技能包
- 与scientific-agent-skills的147个技能兼容

### 6. AutoResearchClaw的反幻觉引用验证（参考可信度）

4层引用完整性验证：
- arXiv/CrossRef/DataCite/LLM交叉验证
- 自动删除虚假引用
- Inline fact-checking

**蜂群文献蜂当前无引用验证。** 这是最容易实现且最有价值的参考：
- 文献蜂输出后加一步验证
- 用Semantic Scholar API验证DOI真实性
- 标记无法验证的引用

### 7. RD-Agent的MLE-bench领先（参考评测）

RD-Agent在MLE-bench 75个Kaggle竞赛上排名第一（30.22%），有ICML 2026/ACL 2026/NeurIPS 2025论文。

**蜂群没有benchmark。** 可参考：
- 建立蜂群评测基准（类似ARC-Bench的55主题）
- 先在化学领域建立评测（竞品空白）
- 发预印本建立学术背书

## 三、3个不要参考的点

### 1. ❌ 不要照搬23阶段流水线
AutoResearchClaw的23步太重，蜂群优势是轻量。7蜂保持不变，加阶段标记即可。

### 2. ❌ 不要做ACP兼容
AutoResearchClaw支持6种Agent CLI（Claude Code/Codex/Copilot等），但蜂群是SaaS平台，不需要兼容其他Agent。

### 3. ❌ 不要做开源
竞品全开源，蜂群的差异化就是SaaS付费。开源会直接丧失商业模式。

## 五、战略层面的关键启示

### 1. 蜂群的真正对手不是AI-Scientist，是K-Dense

K-Dense scientific-agent-skills（28.7k stars）+ K-Dense BYOK（免费桌面端）的组合：
- 同样走BYOK路线
- 同样覆盖化学/生物/医学
- 同样定位"AI co-scientist"
- **但完全免费开源**

蜂群的SaaS+积分制 vs K-Dense的免费开源，是直接竞争。

### 2. 蜂群必须回答一个问题：为什么付费？

当AutoResearchClaw（23阶段全自动+Co-Pilot）、ARIS（79技能+跨模型评审）、K-Dense（147技能+BYOK）都免费时，蜂群凭什么收钱？

可能答案：
- **A. 化学虚拟实验室深度** — 竞品只有技能接口，蜂群做端到端化学实验（反应预测→逆合成→性质计算→量子化学）
- **B. 零部署门槛** — 竞品需自部署（Docker/Python/Claude Code），蜂群即开即用
- **C. 算力包含** — 竞品BYOK需自己有API key+GPU，蜂群积分含算力

### 3. 建议的差异化定位

```
蜂群 = 化学虚拟实验室SaaS
不是：通用AI科研工具（太多竞品）
而是：化学研究者的专用平台
```

化学方向是竞品最大空白：
- AI-Scientist/RD-Agent/AI-Researcher：仅ML
- AutoResearchClaw：有化学Docker但浅尝
- InternAgent：涉及化学但非核心
- K-Dense：有化学技能但不端到端
- ARIS：无化学

**蜂群应该All in化学。**
