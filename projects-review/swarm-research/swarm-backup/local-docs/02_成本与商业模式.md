## 一、算力引擎怎么接：3种模式

### 模式A：本地引擎（零成本，先做这个）

| 引擎 | 运行方式 | 成本 | 精度 |
|------|----------|------|------|
| RDKit | Python库，CPU | ¥0 | 高（确定论） |
| OpenMM | Python库，CPU/GPU | ¥0 | 高（物理引擎） |
| PySCF | Python库，CPU | ¥0 | 最高（量子化学） |
| Docker沙箱 | 本地容器 | ¥0 | 真实执行 |

**接入方式：pip install → Python直接调用 → 蜂群Agent写代码调用**

```python
# 化学蜂调用RDKit示例
from rdkit import Chem
from rdkit.Chem import AllChem

mol = Chem.MolFromSmiles('CCO')
AllChem.EmbedMolecule(mol)
# 计算3D结构、能量、性质...
```

**优点：零成本，已有服务器就能跑**
**缺点：CPU慢，大分子模拟可能要几小时**

### 模式B：第三方API（低成本，按需）

| API | 免费额度 | 超额价格 | 用途 |
|-----|----------|----------|------|
| IBM RXN | 免费注册 | 按次计费 | 反应预测、逆合成 |
| Semantic Scholar | 无限免费 | — | 文献检索 |
| PubChem | 无限免费 | — | 分子数据 |
| AlphaFold3 | 开源本地跑 | ¥0 | 蛋白结构 |

**接入方式：HTTP API调用，蜂群Agent按需请求**

**优点：免费额度够用，无需算力**
**缺点：依赖外部服务，有限额**

### 模式C：按需GPU（高成本，最后考虑）

| 场景 | 何时需要 | 单次成本 |
|------|----------|----------|
| ML模型训练 | 实验蜂跑深度学习 | ¥5-15/小时 |
| 大分子动力学 | 10000+原子模拟 | ¥10-25/小时 |
| 量子化学大体系 | 100+原子精确计算 | ¥15-50/小时 |

**接入方式：用户付费→平台按需租GPU→用完释放**

**优点：能跑大实验**
**缺点：贵，只有付费用户才用**

平台     → 只赚差价（GPU加价20%）

## 四、成本控制5个铁律

### 1. 免费用户只用CPU引擎
- 不给免费用户GPU权限
- CPU跑RDKit/OpenMM够用（小分子<500原子）
- 免费用户单次实验限制5分钟CPU

### 2. LLM用便宜模型
- GLM-4-Air（¥0.5/M tokens）代替GLM-4-Plus（¥5/M）
- 比特助手（免费）做简单任务：摘要、格式化、分类
- 只有关键推理（假设生成）用GLM-4-Plus

### 3. 第三方API用免费额度
- IBM RXN：免费额度够100次/月
- Semantic Scholar：完全免费
- 不调付费API

### 4. GPU按需租用，不囤卡
- 用户付费才租GPU
- 平台不买GPU卡
- 租用差价20%

### 5. 用户自带算力模式
- 企业版支持BYOK（Bring Your Own Key）
- 用户填自己的GLM-4 API Key
- 用户填自己的GPU服务器
- 平台只收软件费
