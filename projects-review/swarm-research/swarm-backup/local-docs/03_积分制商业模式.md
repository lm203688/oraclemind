## 四、对比传统定价的优势

| 维度 | 传统月费制 | 积分制 |
|------|-----------|--------|
| 用户心理 | "¥39/月"感觉贵 | "440积分"感觉多 |
| 现金流 | 后付费，月结 | **预充值，先收钱** |
| 空置率 | 用户不充钱也能用 | **不用就过期，促活** |
| 弹性定价 | 改价格用户敏感 | **调积分消耗，用户不敏感** |
| 促销 | 降价影响品牌 | **送积分，不影响定价** |
| 成本控制 | 固定月费风险 | **用多少扣多少，零风险** |
| 模块化 | 难拆分 | **天然模块化，每模块独立扣分** |

## 六、技术实现

### 积分系统API

```python
# 积分账户
POST /api/v1/credits/purchase      # 充值
GET  /api/v1/credits/balance        # 查余额
POST /api/v1/credits/spend          # 消耗（内部调用）
GET  /api/v1/credits/history        # 消费记录
POST /api/v1/credits/redeem         # 兑换码

# 蜂群任务
POST /api/v1/swarm/start            # 启动蜂群（预估积分）
GET  /api/v1/swarm/{id}/status      # 查看进度+已消耗积分
GET  /api/v1/swarm/{id}/result      # 获取结果

# 模块管理
POST /api/v1/modules/unlock         # 解锁模块
GET  /api/v1/modules/owned          # 已解锁模块
```

### 积分消耗流程

```
用户启动任务 → 预估积分 → 冻结积分 → 执行蜂群 → 实际消耗 → 退回差额
```

**防止超额消费：预扣机制，余额不足则拒绝执行。**
