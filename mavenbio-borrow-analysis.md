# MavenBio模式借鉴分析 — GeneTech 14站优化方案

> 2026-06-25 | 基于MavenBio(YC S23)产品拆解

## 一、MavenBio四大模块 vs 我们的差距

### MavenBio产品架构

| 模块 | 功能 | 定价价值 | 我们的现状 | 差距 |
|------|------|---------|-----------|------|
| **Maven Workflows** | AI分步执行研究任务，可追踪可审计 | 核心卖点，高客单价 | ❌ 完全没有 | **最大差距** |
| **Maven Atlas** | 预建适应症全景图 + 按需生成 | 高价值，差异化 | ⚠️ 有数据无全景视角 | 中等 |
| **Maven Compass** | 精确筛选公司/药物/试验，AI推荐 | 实用工具，粘性 | ⚠️ 有实体筛选弱 | 中等 |
| **Maven Assistant** | 对话式问答 + 引用原始来源 | 入口级，低门槛 | ⚠️ 有API无对话 | 小 |

## 二、可直接拿来用的4个模式

### 1. Workflow模板层 → 最高优先级

**MavenBio怎么做：**
用户提出研究问题（如"评估靶点X在自身免疫疾病中的商业潜力"），AI自动拆解为多步骤：查靶点机制 → 查在研药物 → 查临床试验 → 查竞品 → 查专利 → 汇总报告。每步可追踪、可审计。

**我们怎么搬：**
在现有Agent API上加一层Workflow模板。每个站点预设3-5个工作流，用户选择后AI自动调用API分步执行。

**每站工作流设计（示例）：**

| 站点 | 工作流 | 步骤 |
|------|--------|------|
| genetech-tools | 基因治疗管线评估 | 查靶点→查疗法→查公司→查临床→输出报告 |
| genetech-tools | CRISPR应用可行性分析 | 查靶基因→查编辑类型→查递送方式→查竞品→输出报告 |
| tcm-tools | 中药方剂循证分析 | 查药材→查活性成分→查靶点→查临床证据→输出报告 |
| tcm-tools | 中药新药审批追踪 | 查品种→查分类→查临床试验→查审批状态→输出报告 |
| quantum-computing | 量子算法应用评估 | 查算法→查硬件需求→查竞品→查成熟度→输出报告 |
| brain-science | 脑疾病治疗方案全景 | 查疾病→查靶点→查疗法→查临床试验→输出报告 |
| bionic-ai | 仿生技术商业化评估 | 查技术→查公司→查应用场景→查市场规模→输出报告 |
| ... | ... | ... |

**技术实现：**
- API: `POST /api/v1/workflow/{site}/{template_id}` → 返回workflow_run_id
- 异步执行，每步结果可通过 `GET /api/v1/workflow/run/{id}` 查询
- 最终输出结构化JSON报告 + Markdown摘要

**工作量估算：** 3-5天（14站×3-5个模板=42-70个工作流定义）

### 2. Atlas全景图 → 每站核心页面

**MavenBio怎么做：**
预建数百个"适应症全景图"——用户打开即看到某疾病领域完整的管线分布、靶点分布、公司分布。也可按需生成。

**我们怎么搬：**
每个站点按核心维度生成"全景图"页面——不是新数据，是现有数据的聚合视图。

**示例：**

| 站点 | 全景图 | 聚合维度 |
|------|--------|---------|
| genetech-tools | 基因治疗管线全景 | 按疾病→疗法类型→开发阶段→公司 聚合 |
| genetech-tools | CRISPR应用全景 | 按编辑类型→靶基因→疾病→阶段 聚合 |
| tcm-tools | 中药创新药全景 | 按分类→适应症→公司→审批日期 聚合 |
| brain-science | 阿尔茨海默症全景 | 按靶点→疗法→临床阶段→公司 聚合 |
| bionic-ai | 神经形态计算全景 | 按技术类型→公司→应用→成熟度 聚合 |
| new-energy | 固态电池全景 | 按材料→公司→技术路线→成熟度 聚合 |

**技术实现：**
- 每站新增 `/atlas/` 目录，生成静态HTML全景图页面
- 数据从现有data.json聚合，无需新增数据
- 支持按需生成：`POST /api/v1/atlas/{site}/{topic}` → 返回聚合JSON

**工作量估算：** 2-3天（14站×3-5个全景图）

### 3. Compass智能筛选 → API增强

**MavenBio怎么做：**
精确筛选公司/药物/试验/文档，AI推荐筛选条件。比如用户选"PD-1"后，AI建议"是否同时筛选正在招募的III期试验"。

**我们怎么搬：**
现有API已支持基础查询，加两层：
- **高级筛选API**: `GET /api/v1/search/{site}?filters={field:value,field:value}&sort=...&limit=...`
- **AI推荐筛选**: 基于当前查询结果，LLM分析后推荐"你可能还想筛选..."

**技术实现：**
- 扩展现有API，支持多字段AND/OR筛选
- AI推荐用轻量LLM调用（基于结果字段分布生成建议）
- 前端加筛选面板（可后续做）

**工作量估算：** 1-2天

### 4. Token计量体系 → 为用量制定价铺路

**MavenBio怎么做：**
用量制定价 + 试用限制。Token用量55.6B说明有大量真实消费。

**我们怎么搬：**
API层加usage tracking中间件：
- 每次API调用记录：API Key + 端点 + 返回实体数 + Token消耗
- `GET /api/v1/usage?key=gtk_xxx` → 返回用量统计
- Free tier: 1000次/月, Pro tier: 50000次/月, Team: 无限

**工作量估算：** 1天

## 三、模式优化（不是照搬，是改进）

### 优化1：多站联动 > 单站专注

MavenBio只做BioPharma一个领域。我们14站可以**跨站联动工作流**：

- "基因治疗→中医药跨界分析"：查genetech基因靶点 → 查tcm是否有对应活性成分 → 输出中西医结合报告
- "脑科学→仿生AI交叉"：查brain-science脑机制 → 查bionic-ai是否有仿生应用 → 输出交叉创新报告
- "新能源→外星矿物"：查new-energy材料需求 → 查alien-minerals是否有太空矿物匹配 → 输出太空采矿+新能源报告

**这是MavenBio做不到的——他们只有一个领域。**

### 优化2：PLG自助 > 企业销售

MavenBio 8人团队做企业销售，我们1人+AI做PLG（产品驱动增长）：
- Free tier通过API自助接入 → 用量到上限 → 自动引导付费
- 不需要销售团队，产品本身就是销售
- Creem支付已就绪，6档定价已上线

### 优化3：Agent原生分发 > 传统获客

MavenBio靠YC品牌+企业销售获客。我们走Agent原生分发：
- llms.txt + schema.org + MCP Server → AI Agent直接发现并调用我们的API
- 14站每天IndexNow提交 → SEO流量
- GitHub awesome lists PR → 开发者社区引流

## 五、不建议借鉴的

- **专注单领域**：我们14站广度是壁垒，不收缩
- **企业销售模式**：1人团队撑不起8人销售模式
- **LlamaParse依赖**：自建解析能力更可控
- **高客单价咨询**：我们走API自助路线，定价从$19/月起
