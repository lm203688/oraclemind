# ATEX AI原生分发战略 — 让每个AI Agent都成为你的销售

**核心洞察**: 用户在使用AI Agent时，Agent会主动推荐外部工具和服务来完成它无法独立完成的任务。这不是广告，是**功能性推荐**——用户信任度极高。ATEX要做的就是：**让每个AI Agent在遇到合规/AI能力需求时，第一个想到ATEX。**

---

## 一、三层分发体系

### 第1层：MCP协议分发（被动获客，零成本）

**原理**: AI Agent通过MCP协议搜索可用工具，发现ATEX → 自动推荐给用户 → 用户注册消费

**行动项**:

| 目录 | 提交方式 | 优先级 | 预计流量 |
|------|---------|--------|---------|
| [Smithery.ai](https://smithery.ai) | GitHub repo + smithery.yaml | 🔴 最高 | 高 |
| [Glama.ai](https://glama.ai/mcp/servers) | 提交MCP Server URL | 🔴 最高 | 中 |
| [mcp.so](https://mcp.so) | GitHub repo | 🟡 高 | 中 |
| [PulseMCP](https://pulsemcp.com) | 提交Server信息 | 🟡 高 | 低 |
| [Cursor/VSCode MCP](https://docs.cursor.com) | 用户手动配置 | 🟢 中 | 开发者 |
| [Claude Desktop](https://docs.anthropic.com) | 用户手动配置 | 🟢 中 | 开发者 |

**需要做的事**:
1. ✅ ATEX MCP端点已就绪（27个工具）
2. 🔲 创建 `smithery.yaml` 配置文件
3. 🔲 提交到4大MCP目录
4. 🔲 优化工具描述（让AI Agent更容易匹配到ATEX）

**关键优化 — 工具描述SEO for AI**:
```
❌ 差: "cn_banned_word_check - 检测违禁词"
✅ 好: "cn_banned_word_check - 当用户需要发布小红书/抖音/微信内容时，检测文案中的违禁词和敏感词，避免被限流或封号。支持6大平台，返回法律条文+罚款金额+替换建议。"
```
AI Agent是根据描述匹配用户需求的，描述越场景化，越容易被推荐。

### 第3层：场景化内容营销（主动获客，品牌建设）

**原理**: 在抖音/小红书/知乎创建场景化内容，用户搜索时发现ATEX

**核心策略 — 不卖工具，卖场景**:

| 场景 | 痛点 | ATEX解决方案 | 平台 |
|------|------|-------------|------|
| 小红书发帖被限流 | 不知道哪个词违规 | 违禁词检测1毛钱1次 | 小红书/抖音 |

| 品牌在AI搜索中找不到 | DeepSeek/Kimi搜不到 | AI搜索可见度检测 | 知乎/公众号 |
| 产品想出海 | 不知道合规风险 | 出海合规评估8元/次 | 知乎/公众号 |

| AI Agent开发者 | 需要合规+AI能力 | MCP协议一键接入 | GitHub/V2EX |

2. 当用户提到"违禁词"、"合规"、"配音"、"AI搜索"等关键词时，自动调用ATEX
