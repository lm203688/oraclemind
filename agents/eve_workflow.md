# Eve总管 — Agent团队工作流规范

> 用户确认：2026-07-02 03:34
> 状态：正式执行

## 一、每日任务（Daily）

### 1. 运营财务官 — 每日监控（最先执行）
- 时间：每日08:00
- 任务：监控所有项目服务状态+收入+成本
- 输出：异常清单→交给Eve
- Eve判断：bug→工程师，营销问题→营销师，其他→对应agent

### 2. 情报员 — 14站情报收集
- 时间：每日08:30（运营财务官之后）
- 任务：14站逐站采集市场信息+新增市场反馈
- 输出：每日情报简报→交给Eve→分发给分析师/营销师

### 3. 工程师 — 执行修改任务
- 时间：每日09:00-22:00（按Eve分发的任务）
- 任务：修复bug+功能改进+营销师编程需求
- 输出：修改清单→交给审计师

### 4. 审计师 — 每日全面审计
- 时间：每日22:00（工程师完成修改后）
- 任务：审计当日修改+链路通畅+法律合规
- 输出：审计报告→交给Eve→安排对应agent修复

### 5. Eve — 每日报告
- 时间：每日23:00
- 任务：汇总所有agent当日工作→生成项目情况清单
- 输出：发送给用户，含需用户回答的疑问

## 二、每周任务（Weekly）

### 1. 情报员 — 非14站项目情报
- 时间：每周一08:30
- 任务：收集aishield/roboparts/swarm-research/healthlens/agent-trust的市场信息
- 输出：周度情报报告→交给分析师

### 2. 分析师 — 周度改进建议
- 时间：每周二08:00（情报员之后）
- 任务：根据情报+项目运行情况→提出改进建议
- 输出：改进建议清单→交给Eve→判断后交工程师

### 3. 营销师 — 周度营销执行
- 时间：每周三08:00（分析师之后）
- 任务：根据情报+分析+运营反馈→执行营销任务
- 输出：营销执行报告→交给Eve→调整策略
- 需工程师配合的编程任务→Eve转交工程师

## 三、每月任务（Monthly）

### 1. 设计师 — UI设计审查
- 时间：每月1日08:00
- 任务：整体UI设计+本月修改处的设计建议
- 输出：设计建议→交给Eve→交工程师改进

### 2. 研究师 — 前沿研究
- 时间：每月1日10:00（分析师报告之后）
- 任务：分析师报告中提到的前沿方向→研究论证
- 输出：前沿观点/结论→经论证后交Eve→放入项目

### 3. Eve — 月度备份
- 时间：每月1日23:00
- 任务：打包所有项目资料→压缩包→发送用户备案

## 四、Eve调度逻辑

```
每日流程：
08:00 运营财务官监控 → 异常清单
08:30 情报员14站采集 → 情报简报
09:00 Eve分析异常+情报 → 分发任务
      ├→ 工程师：修复bug/功能改进
      ├→ 营销师：调整推广
      └→ 分析师：（按需）
22:00 审计师全面审计 → 审计报告
23:00 Eve汇总 → 每日报告发送用户

每周流程：
周一 08:30 情报员非14站项目情报
周二 08:00 分析师周度改进建议
周三 08:00 营销师周度营销执行

每月流程：
1日 08:00 设计师UI审查
1日 10:00 研究师前沿研究
1日 23:00 Eve月度备份
```
