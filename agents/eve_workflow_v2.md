## 一、设计原则

1. **Eve是唯一枢纽** — 所有Agent间通信通过Eve，Agent之间零耦合
2. **任务优先级驱动** — P0立即/P1当日/P2本周/P3月度
3. **情报看板共享** — 所有Agent可读，情报员写入
4. **失败自动降级** — LLM超时降级模型，连续失败标记人工介入
5. **成本意识贯穿** — 每个Agent记录消耗，运营财务官月度汇总

---

## 二、任务优先级

| 优先级 | 含义 | 响应时间 | 示例 |
|---|---|---|---|
| **P0** | 线上故障 | 立即处理+Bark推送 | 服务宕机、支付失败 |
| **P1** | 当日修复 | 当日22:00前 | critical安全漏洞、营销改动需工程配合 |
| **P2** | 本周修复 | 本周内 | 分析师改进建议、功能优化 |
| **P3** | 月度排期 | 本月内 | 设计师UI建议、非紧急功能 |

### 异常升级路径
```
critical → 2小时内通知用户 + Bark推送
high → 当日修复
medium → 本周修复
low → 月度排期
```

## 三、每日任务（Daily）

### 时间表

| 时间 | Agent | 任务 | 输出 |
|---|---|---|---|
| 08:00 | 运营财务官 | 全项目服务监控+收入+成本 | 异常清单（按P0/P1分级）→ Eve |
| 08:30 | 情报员 | 14站逐站采集市场信息+市场反馈 | 情报简报 → 情报看板 + Eve |
| 09:00 | Eve | 分析异常+情报 → 分发任务（按优先级） | 任务清单分发到各Agent |
| 09:00-22:00 | 工程师 | 执行任务（P0>P1>P2>P3） | 修改清单 → 审计师 |
| 22:00 | 审计师 | 审计当日修改+依赖库检查+链路通畅 | 审计报告 → Eve |
| 23:00 | Eve | 汇总所有Agent当日工作 → 生成日报 | 发送用户（含需决策项） |

### P0故障处理流程
```
运营财务官发现P0
  → Bark立即推送到手机
  → Eve同步通知工程师
  → 工程师立即处理
  → 修复后审计师验证
  → Eve确认恢复 → 通知用户
```

### 营销改动需工程配合
```
Eve判断营销策略需要代码改动
  → 标记为P1（当日修复）
  → 交工程师处理
  → 完成后交营销师执行
```

## 四、每周任务（Weekly）

### 时间表

| 星期 | 时间 | Agent | 任务 | 输出 |
|---|---|---|---|---|
| 周一 | 08:30 | 情报员 | 非14站项目情报（aishield/roboparts/swarm/healthlens/agent-trust） | 周度情报报告 → 看板 + 分析师 |
| 周一 | 10:00 | 科研员 | 轻量前沿扫描（各项目相关领域） | 前沿简报 → Eve（重大发现触发深度论证） |
| 周二 | 08:00 | 分析师 | 深度分析+改进建议（基于情报+运行数据） | 改进建议清单 → Eve → 工程师 |
| 周二 | 10:00 | 科研员 | 深度论证（如有重大前沿发现） | 论证报告 → Eve → 放入项目 |
| 周三 | 08:00 | 营销师 | 执行营销任务+上周效果报告 | 营销执行报告 → Eve → 调整策略 |
| 周四 | 08:00 | 审计师 | 全量安全扫描（所有项目） | 安全报告 → Eve |

### 情报员每周新增站点评估
```
情报员在周报中提出新站建议
  → 分析师评估商业价值
  → Eve判断
  → 用户确认
  → 工程师搭建
```

## 五、每月任务（Monthly）

### 时间表（每月1日）

| 时间 | Agent | 任务 | 输出 |
|---|---|---|---|
| 08:00 | 设计师 | 整体UI审查+本月修改处设计建议 | 设计建议 → Eve → 工程师 |
| 10:00 | 分析师 | 月度商业复盘（收入/转化/ROI） | 商业报告 → Eve |
| 14:00 | 科研员 | 月度前沿深度报告 | 研究报告 → Eve |
| 18:00 | 运营财务官 | 月度成本+各Agent消耗汇总 | 成本报告 → Eve |
| 22:00 | 审计师 | 法律合规全量审计 | 合规报告 → Eve |
| 23:00 | Eve | 月度备份+验证+月报 | 压缩包发送用户 |

### 月度备份内容
- 所有项目代码（agents/ + 项目源码）
- 所有配置文件（TOOLS.md/AGENTS.md/SOUL.md等）
- 所有Agent数据库（db/）
- 所有Agent项目记忆（project_memory/）
- 所有报告（reports/）
- 情报看板数据

### 备份验证
- 文件数校验
- 关键文件存在性检查
- 压缩完整性验证
- 验证失败→Bark推送告警

## 六、情报看板

### 位置
`/home/z/my-project/agents/shared/intel_board.json`

### 结构
```json
{
  "last_updated": "2026-07-02T08:30:00",
  "entries": [
    {
      "id": "intel-001",
      "date": "2026-07-02",
      "source": "scout",
      "project": "genetech-tools",
      "category": "market_trend",
      "title": "CRISPR个性化治疗突破",
      "content": "...",
      "impact": "high",
      "actions_suggested": ["更新知识库CRISPR实体"],
      "read_by": []
    }
  ]
}
```

### 权限
- **情报员**：写入（唯一写入者）
- **其他Agent**：只读

### 看板维护
- 情报保留30天，超期自动归档
- Eve每日清理已读且无action的条目

## 八、失败处理与降级机制

### LLM调用失败
```
glm-4-plus超时 → 重试1次
  → 仍失败 → 降级到glm-4-flash
  → 仍失败 → 标记"LLM不可用"，Eve决定是否等用户
```

### web_search失败
```
搜索失败 → 换关键词重试
  → 仍失败 → 用14站知识库兜底
  → 仍失败 → 标记"情报采集受限"，日报中说明
```

### Agent服务失败
```
Agent执行失败 → 重试1次
  → 仍失败 → Eve记录到growth-log
  → 连续失败3次 → 标记"需人工介入"，Bark推送
```
