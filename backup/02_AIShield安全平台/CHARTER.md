# AIShield 项目纲领
# 创建: 2026-06-23
# 状态: 生效中

## 北极星

**AI Agent可信生态的安全基础设施**

让每个Agent都有安全背书，让每次Agent调用都可信、可计费、可追溯。

## 三阶段路径

### 阶段1: 合规安全切入 (0-3个月 | 2026Q3)
**目标: 产生现金流，积累用户和安全数据**

主打产品:
- 违禁词检测 ¥0.5/次 (引流)
- MCP/Agent安全扫描 ¥10/次 (差异化)
- 出海合规评估 ¥50/次 (高客单价)
- SEO合规检测 ¥3/次 (增值)
- AI搜索可见度检测 ¥5/次 (品牌监测)

收入模型:
- 免费层: 每天3次违禁词检测 (引流)
- 按次付费: ¥0.5-50/次
- 月度订阅: ¥99/月 = 200次违禁词 + 50次AI对话 + 10次SEO合规
- 企业定制: ¥999/月 = 不限次 + API接入 + 专属合规报告

关键指标:
- 月收入 ≥ ¥3000
- 注册用户 ≥ 200
- 违禁词检测调用量 ≥ 1000次/月
- MCP安全扫描工具数 ≥ 100个

### 阶段2: Agent可信工具市场 (3-6个月 | 2026Q4)
**目标: 建立Agent安全信任壁垒**

核心动作:
- 扫描所有公开MCP Server (持续扩展)
- 给安全评分 (A/B/C/D)
- 优秀工具获得"AIShield认证"标签
- 用户通过平台调用认证工具 (赚调用差价)

竞争壁垒:
- Smithery只列工具，AIShield给工具做安全背书
- 安全数据是飞轮: 扫描越多→数据越全→信任越高→用户越多→工具越多

### 阶段3: Agent经济闭环 (6-12个月 | 2027Q1-Q2)
**目标: 成为Agent生态的安全层+支付层**

闭环:
```
Agent开发者 → 提交工具 → AIShield安全审计 → 认证标签
Agent用户 → 发现可信工具 → 通过平台调用 → 自动计费
数据源(12站) → Agent调用知识API → 按次计费
```

## 决策规则 (遇到选择时参考)

1. **新功能是否推进?** → 是否服务于"Agent安全"或"合规工具"? 否则不推进
2. **定价怎么定?** → 参考人工替代成本，定价为其1/10-1/100
3. **要砍什么?** → 与"AI合规+Agent安全"无关的功能，砍
4. **优先做什么?** → 能产生现金流+积累安全数据的优先
5. **品牌怎么延伸?** → 一切以"AIShield=安全背书"为核心延伸

## 禁止事项

- ❌ 不做通用AI API网关 (红海，打不过OpenRouter)
- ❌ 不做MCP目录 (打不过Smithery)
- ❌ 不做与合规安全无关的功能
- ❌ 不把AI能力当主打 (只做增值upsell)
- ❌ 不用"ATEX"或"Exchange"作为品牌名
